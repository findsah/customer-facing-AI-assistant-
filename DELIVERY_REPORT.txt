7. TECHNOLOGY CHOICES EXPLAINED

Web Scraping
  Library: BeautifulSoup + requests
  Why: Simple, lightweight, perfect for static HTML
  Alternatives: Selenium (for JS), Scrapy (for complex crawling)

Embeddings
  Model: all-MiniLM-L6-v2 (HuggingFace)
  Why: Open-source, fast (384 dims), high quality for domain
  Cost: Free (local) vs $0.02/1K tokens (OpenAI)
  Alternative: text-davinci-003, Cohere API

Vector Store
  Database: Chroma
  Why: Local, simple, persistent, no external dependencies
  Scales to: 1M+ vectors easily
  Alternative: Pinecone (cloud), Weaviate (complex setup)

LLM Options
  Local: Mistral-7B (open-source, 14GB)
  API: OpenAI GPT-4 (requires key)
  Fallback: Simple retrieval-only mode (always works)

API Framework
  Framework: FastAPI
  Why: Modern, async, auto-documented, Pydantic validation
  Performance: 3x faster than Flask
  Alternative: Django REST Framework (heavier)

Containerization
  Tools: Docker + Docker Compose
  Why: Simple for prototyping, easy AWS migration
  Alternative: Kubernetes (for enterprise scale)

Database
  Primary: Chroma (vectors)
  Metadata: SQLite (in Chroma)
  Production: Could add RDS PostgreSQL

8. AWS DEPLOYMENT ARCHITECTURE

Recommended AWS Services:

Frontend
  - CloudFront CDN (caching, DDoS protection)

API Layer
  - API Gateway or ALB (routing, rate limiting)
  - ECS Fargate (container orchestration, auto-scaling)

Data Layer
  - OpenSearch (distributed vector store)
  - ElastiCache Redis (caching)
  - RDS Aurora PostgreSQL (metadata)
  - S3 (backups, data lake)

Operations
  - Lambda (scheduled scraping)
  - CloudWatch (monitoring)
  - SNS (alerting)
  - SageMaker (optional: fine-tune models)

Scaling Strategy:
  1-100 users:   1 ECS task (4GB mem, 2 CPU)
  100-1000:      3-5 ECS tasks (auto-scaling)
  1000-10000:    10-20 ECS tasks + managed OpenSearch
  10000+:        Kubernetes on EKS + multi-region

  Prod (10000 users): ~$2000/month
